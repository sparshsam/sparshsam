# Publication Roadmap

## 2026: Foundation

- Create profile repository and identity documentation.
- Standardize repository metadata, topics, badges, and descriptions.
- Add `CITATION.cff` and `.zenodo.json` to mature repositories.
- Prepare DOI release candidates for OpenProof, QuietLedger, and PDFReader.
- Draft the first architecture notes.

## 2027: Architecture Papers

- OpenProof: proof-of-existence for personal records.
- QuietLedger: local-first finance without behavioral capture.
- PDFReader: private document workspaces and local annotation continuity.
- Elora Vault: behavioral savings without gamification.

## 2028: RFC Layer

- RFC 0001: Calm Interface Checklist.
- RFC 0002: Personal Proof Record Format.
- RFC 0003: Local Finance Export Schema.
- RFC 0004: Minimal Disclosure Civic Report Model.

## 2029-2030: Reference Implementations

- Mature OpenProof into a reference implementation for personal proof records.
- Publish interoperable export/import schemas for QuietLedger and OpenSprout.
- Create cross-project test fixtures and archival datasets where privacy permits.

## 2031-2036: Research Program

- Annual calm infrastructure report.
- Protocol specifications with DOI-indexed versions.
- Comparative studies of local-first patterns in personal software.
- Public maintenance notes on long-lived independent open-source systems.

## Output Types

- whitepapers;
- RFCs;
- architecture papers;
- protocol specifications;
- threat models;
- privacy models;
- release reports;
- annotated reference implementations.
