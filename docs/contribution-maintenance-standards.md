# Contribution and Maintenance Standards

## Maintainer Posture

The ecosystem should welcome useful contribution without turning maintenance into performance. The tone should be serious, calm, and bounded.

## Required Standards

Every mature repository should include:

- issue templates;
- pull request template;
- code of conduct;
- security policy;
- contributing guide;
- changelog;
- citation metadata;
- release process;
- privacy model.

## Contribution Rules

- Contributions should preserve privacy-first defaults.
- Features that increase behavioral pressure require explicit justification.
- New telemetry is rejected unless it is local, optional, documented, and easy to disable.
- Export and deletion paths must remain visible.
- Dependencies should be justified when they affect custody, build reproducibility, or security.
- UI changes should preserve calmness, legibility, and accessibility.

## Review Checklist

- Does this change alter what data is stored?
- Does this change alter where data leaves the user's device?
- Does this introduce urgency, compulsion, shame, or artificial scarcity?
- Is the behavior documented?
- Are tests or manual verification steps included?
- Can a user still export or recover their data?

## Release Checklist

- Version tag created.
- Changelog updated.
- README status accurate.
- Citation metadata current.
- Zenodo metadata current.
- Known limitations documented.
- Security notes reviewed.
