# Lightweight Visual Identity

The visual identity should be quiet, durable, and documentation-friendly.

## Design Principles

- Prefer typography and spacing over illustration.
- Use muted color sparingly.
- Avoid gradients, hype visuals, mascot language, and dashboard theatrics.
- Treat diagrams as infrastructure maps, not promotional art.
- Keep badges flat and restrained.

## Palette

Use a small neutral palette with one green-gray accent and one blue-gray accent:

- Ink: `#1f2933`
- Slate: `#44546a`
- Mist: `#e8edf1`
- Paper: `#f8faf9`
- Moss: `#2f6f5e`
- Signal: `#3b6f8f`

## Typography Direction

- Documentation: system sans or Inter.
- Code and specs: system mono, JetBrains Mono, or Geist Mono.
- Avoid expressive display type for technical docs.

## Diagram Style

- White or paper background.
- Thin lines.
- Short labels.
- No 3D effects.
- No decorative shadows.

## README Style

- One concise opening paragraph.
- Project metadata block near the top.
- Architecture and privacy sections before screenshots.
- Screenshots only when they reveal actual product state.
