# Contributing

Thank you for considering a contribution.

This project is part of a calm, privacy-first open-source ecosystem. Contributions should preserve user custody, clear documentation, and emotionally restrained design.

## Principles

- Keep privacy-first defaults intact.
- Avoid telemetry unless it is local, optional, documented, and easy to disable.
- Avoid engagement mechanics such as streak pressure, shame states, artificial urgency, or manipulative rewards.
- Prefer clear, maintainable code over clever abstractions.
- Document changes that affect data storage, export, deletion, or verification.

## Pull Requests

Before opening a pull request:

- describe the user-facing change;
- include tests or manual verification steps;
- note any privacy, security, or data-custody impact;
- update documentation when behavior changes.
